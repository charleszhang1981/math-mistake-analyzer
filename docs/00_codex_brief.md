# 数学纠错本项目简介（现状版）

> 更新时间：2026-03-06
> 口径：以当前仓库实现为准，不再沿用早期 MVP 设想中的未落地方案。

## 1. 项目定位

这是一个以 **数学纠错本** 为主场景的 Next.js 应用，核心目标是把“拍题/截图后的错题整理、结构化分析、复习、再练习、打印导出”串成一个可用闭环。

当前产品叙事应分为两层：

- **主场景**：数学单错题本、PC-first、面向个人/家庭使用的纠错与复习流程。
- **平台能力**：代码中已经具备多用户注册登录、管理员后台、统计、打印、练习生成等通用能力，但这些能力仍服务于“数学锁定”的当前产品路径。

## 2. 当前核心闭环

当前已落地的主流程是：

1. 上传图片、拍照或截图
2. 原图上传到 Supabase Storage
3. 进入单题裁剪界面，生成裁剪图
4. 将裁剪图送入 AI 两阶段分析
5. 生成并编辑 `structuredJson v2`
6. 用户人工校正 G/H/I 信息后保存为错题
7. 在错题详情、复习、练习、打印页面继续使用

其中 G/H/I 指当前详情页与编辑页围绕的三块核心内容：

- **G 标准解法**：标准答案与分步解法
- **H 错误定位**：学生步骤、首个错误步骤、错误原因、修正建议
- **I 根因自诊断**：用户最终确认的根因

## 3. 已确认的实现边界

### 3.1 数学锁定

- 数据库仍保留 `Subject` / `KnowledgeTag.subject` 等多学科结构。
- 但产品主路径已锁定为数学：
  - 错题本 API 只返回一个 `Math` 错题本
  - 新建/重命名/删除非数学错题本不属于当前允许路径
  - AI 分析、练习生成、标签体系、复习路径都以数学为默认和主线

### 3.2 配置策略：env-only

- 当前配置只从环境变量读取。
- `/api/settings` 的 `GET` 用于给设置页展示当前配置，`POST` 会直接拒绝写入并返回 `CONFIG_ENV_ONLY`。
- 设置页保留了 AI 测试与查看入口，但 **不会** 保存到本地 `config/app-config.json`。

### 3.3 当前 AI 路线

- 当前不是 OCR 服务链路。
- 当前主链路是 **两阶段视觉 LLM**：
  - Stage 1：从图片提取题干、字体大小提示、学生步骤原始文本
  - Stage 2：基于提取结果生成答案、解析、知识点、G/H 字段
- 当前支持的 provider：
  - Gemini
  - OpenAI
  - Azure OpenAI

### 3.4 当前未启用的旧设想

以下内容在代码中要么仅保留字段/工具函数，要么明确未纳入当前主流程：

- OCR 服务接入
- `checkerJson` / `diagnosisJson` 持久化主链路
- 根因聊天接口
- 多学科正式产品路径

## 4. 当前已存在的平台能力

除主纠错链路外，当前仓库还已实现以下能力：

- 用户注册、登录、权限区分
- 管理员用户管理与系统重置
- 题号自动生成
- 错题筛选、批量删除、打印预览
- 复习列表与复习记录
- 基于错题上下文的练习题生成
- 练习统计与错题统计
- 私有图片存储与签名 URL 展示

## 5. 当前最重要的事实纠偏

以下表述不应再出现在设计文档中：

- “当前主链路依赖 OCR”
- “设置支持写本地 config 文件”
- “系统当前支持多学科主路径”
- “checker/diagnosis 已接入错题保存主流程”
- “root-cause chat 当前可直接使用”

正确表述应为：

- 主链路是数学锁定 + 两阶段视觉 LLM
- 配置是 env-only
- 主数据结构是 `structuredJson v2`
- 根因目前以手工填写为主
- 根因聊天接口当前返回 `410`
