# Print Image Scale Persistence Design

## 背景

当前打印预览页里原题图片的展示宽度主要依赖 `fontSizeHint`：

- `large -> 60%`
- `normal -> 80%`
- `small -> 90%`

这个规则是全自动的，但对一些“裁剪后非常扁、文字很小”的题目不够可靠。用户已经明确希望在打印预览页里手动调整每道题的图片比例，并且下次再打开这道题时仍然记住这个比例。

## 目标

- 在打印预览页中为每道题提供原题图片比例调整按钮。
- 调整结果同时作用于：
  - 当前打印预览页面
  - 浏览器打印预览
  - 最终导出的 PDF
- 比例按题持久化保存，下次再打开这道题仍然生效。
- 模式 1 和模式 2 共用同一个比例。

## 方案对比

### 方案 A：新增 `ErrorItem.printImageScale` 字段，推荐

- 在 `ErrorItem` 表上新增数值字段 `printImageScale`。
- 打印页读取该字段作为图片宽度百分比。
- 用户点击 `+ / -` 后立即保存到数据库。

优点：
- 语义清晰，是 UI/打印偏好，不污染 AI 结构化数据。
- 列表接口与打印页天然按题读取，接线最直接。
- 下次打开题目即可复用。

缺点：
- 需要一次数据库迁移。

### 方案 B：把比例写进 `structuredJson`

- 在 `structuredJson.problem` 里新增一个字段。

优点：
- 表面上不用改数据库 schema。

缺点：
- 语义不对，这是打印展示偏好，不是题目内容。
- 容易和 AI/编辑链路混淆。

### 方案 C：只保存在浏览器本地

- 用 `localStorage` 记住题号与比例。

优点：
- 不用改数据库。

缺点：
- 换设备就丢。
- 打印页依赖用户当前浏览器环境，不满足“题目本身记住”的预期。

## 决策

采用方案 A。

## 设计细节

### 1. 数据模型

- 为 `ErrorItem` 新增 `printImageScale Int?`
- 单位直接用百分比整数，例如 `80`
- 默认值保持 `NULL`，表示沿用现有 `fontSizeHint` 推导

### 2. 回退规则

- 若 `printImageScale` 有值，优先使用它
- 若没有值，再回退到现有逻辑：
  - `large -> 60`
  - `normal -> 80`
  - `small -> 90`

### 3. 打印页交互

- 在每道题原图卡片右下角新增 `-` / 当前比例 / `+`
- 只在屏幕上显示，打印与 PDF 内容不显示按钮本身
- 点击后立即更新本地状态并调用保存接口
- 需要有保存中状态，避免重复连点

### 4. 保存方式

- 复用现有 `PUT /api/error-items/[id]`
- 仅提交 `printImageScale`
- 不影响其它字段

### 5. 范围控制

- 模式 1 / 模式 2 共用同一个比例
- 本次不做“重置全部”
- 本次不做拖动滑杆，只做 `+ / -` 小步进按钮

## 验证

- 调整后当前打印页立即生效
- 打开浏览器打印预览时比例一致
- 保存 PDF 时比例一致
- 刷新页面或重新进入打印页后比例仍然保留
